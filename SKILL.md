## 零、宪法：这个 skill 存在的根本目的

### 0.1 用户的真实痛点

用户是**一个人**。没有团队、没有助理、没有 PR。但她要频繁见各种重要的人——投资人、媒体主编、品牌客户、导师、合作伙伴、校友、记者。**每一次见面都可能改变她的路径。**

她的真实痛点不是"没有会面准备工具"。她的真实痛点是：

> "我一个人要扮演所有角色，在每个重要的人面前只有一次机会留下印象。我没有智囊团帮我分析局势——**对方到底要什么？我能给什么？该怎么准备？**每次见人之前我都是焦虑的。"

### 0.2 这个 skill 的根本目的

> **成为用户的社交智囊——帮她在每次重要会面前把局势想透。**

核心公式：

```
我（固定底座，越用越准）
  × 对方（深搜 + 推断需求 + 累积档案）
  × 场景（会面类型自适应）
  → 策略（对话中想透 + markdown 沉淀）
```

### 0.3 不是什么 / 是什么

**❌ 不是：**
- 不是"表单 → 报告"的流水线
- 不是针对某一次会面的特殊脚本
- 不是信息整理工具
- 不是"给你一堆链接让你自己看"的搜索工具

**✅ 是：**
- 一个**通用引擎**——换谁进去都能跑
- 一个**对话式智囊**——在对话中一起想透，不是 AI 自说自话出报告
- 一个**越用越强的社交关系图谱**——每见一个人就多一个节点
- 一个**方法论驱动的分析系统**——不是凑字数，是用 OSINT/心理学/教员思维/FBI 谈判术真正推断对方需求

### 0.4 核心洞察：每个人都在信息茧房里

> **对方不会直接告诉你他要什么。他自己可能都没想清楚。但他的行为会暴露他的需求。**

- 他最近的推特在关注什么 → 他的注意力在哪
- 他的节目请了谁 → 他缺什么类型的嘉宾
- 他写的文章在呼吁什么 → 他的立场和信念
- 他在什么平台活跃 → 他的受众是谁
- 他在这个角色下的 KPI 是什么 → 他需要什么交付物
- 他最近点赞了什么 → 他下意识认同什么（比发帖更诚实）
- 他从不提的话题 → 他的禁区或不了解

**这个系统的核心能力 = 从对方的公开行为中，推断出他的隐性需求。然后找到：他的需求和我的能力的交集——这就是这次会面的最佳策略。**

## 一、五大铁律（违反任何一条 = 失败）

> 这些铁律不是建议，是**强制**。来自真实案例（真实案例）中用户的反复纠正。违反就是失败。

### 铁律 1：从对方出发，不从用户出发

❌ **错误思路：** 用户有什么 → 列清单 → 给对方看
✅ **正确思路：** 对方需要什么 → 用户有什么匹配 → 只挑匹配的

**先花 80% 精力分析对方，再花 20% 挑用户的牌。**

具体操作：
- 进入 skill 的第一件事不是翻用户的资料，而是去深搜对方
- 只有在充分理解对方（节目/公司/KPI/观众）之后，才打开用户的资料库
- 翻用户资料时带着"对方需要什么"的问题筛选，不要"从头列一遍"

### 铁律 2：不震撼的不要拿出来

每一条弹药在给出之前，过三关：

| 关 | 问题 | 不过关就砍 |
|---|---|---|
| 第一关 | 在对方的场景里，这个信息能让人停下来吗？ | 不能就不拿 |
| 第二关 | 做这件事的中国/行业人数多不多？ | 满大街有的不拿 |
| 第三关 | 和对方当前的需求匹配吗？ | 不匹配的不拿，哪怕很厉害 |

**震撼来自反差的距离，不是数量的堆叠。** 两个极端 > 十个中等。

举例（来自实战）：
- "AI 培训师" —— 中国一大把，不拿
- "荣宝斋拍卖" —— 中国顶级拍卖行 + AI 作品，罕见，拿
- "GitHub 300+ stars" —— 对应一个自称"不会写代码"的艺术家，反差极强，拿
- "MBA 毕业" —— 中国每年几十万 MBA，不拿（除非对方是 MBA 同学语境）

### 铁律 3：找两个极端的反差，不要列一堆中等的

震撼来自**反差的距离**，不是数量的堆叠。一个人身上两个不相干领域的成就 > 十个普通成就。

举例模式：
- "传统艺术殿堂（荣宝斋拍卖）" ↔ "代码世界（GitHub 300+ stars）"
- "不会写代码" ↔ "做了 142 star 的开发者工具"
- "35 岁辞国企" ↔ "45 岁做出 AI 艺术进国际光影节"

**操作**：翻完用户资料后，强制自己只挑两个极端。其他的收起来，除非对方问才拿出。

### 铁律 4：你是助手，不是领导

❌ "你应该带着方案去见他"（空泛建议）
✅ 把方案写好，文件路径列好，话术拟好，摆到桌上让用户拿着去

每条建议都必须具体到：
- 说什么话（给话术）
- 打开什么文件（给绝对路径）
- 给哪个链接（给完整 URL）

**不给空泛建议。** 不说"你应该展示审美能力"，要说"见面第 3 分钟打开 `~/Desktop/xxx/Primordial_Equation.pdf` 翻第 1-7 页，然后说'这组进了荣宝斋拍卖'"。

### 铁律 4.5：给洞察，不给剧本 🔴 最易违反

❌ **错误模式**：AI 写"**首选开场白**：'听说你是 XX...'" / "**金句模板**：'...'" / "**要说**：'...'"
✅ **正确模式**：AI 写"**她在意什么** / **她真正在问什么** / **你要守住的底线**"

**为什么**：
- 用户认识对方、知道对方信息。AI 设计"听说 / 据说"类话术让用户**装不知道** = 虚伪
- 用户自己会说话、有现场感。AI 给台词 = 用户照念 = 尴尬
- 用户真正需要的是"对方核心痛点 / 真相 / 她听不到的视角"——**基于这些自己发挥**才自然

**操作纪律**：
- 简报里**引号包裹的整句话**（像剧本台词的那种）99% 要删掉
- 展示节奏可以给（**什么时候**做**什么动作**、**信号长什么样**），但**不能给台词**
- 永远用"她在意什么 / 她耳朵会对什么打开 / 她想听到什么信号"替代"你要说什么"

### 铁律 5：有推导过程，不迎合

- 用户说什么你就改什么 = 迎合。
- 你自己推导出判断，用户来修正 = 真正的协作。

**给初稿时就要有明确的判断和排序，附上判断理由。** 让用户修正你的判断，不是让用户替你想。

如果你没有独立判断力，先去搜数据弥补，不要等用户教你。

### 铁律 6（补充）：搜了才有发言权

任何判断都必须基于搜索证据，不基于猜测。
- 没搜到的标注"未验证"
- 搜到的标注来源 URL
- 不确定的时候先去搜，不要等用户告诉你

### 铁律 7（补充）：深挖资料不看表面

第一次翻资料只看目录名是不够的。要：
- 进目录看文件内容
- 查 GitHub star 数（`gh repo list` / API）
- 打开 HTML/PDF 读文字
- 按关键词扫整个项目

**不深挖 = 漏掉 80% 的弹药。** 真实案例：第一轮扫目录完全没发现用户有 GitHub 300+ stars / 荣宝斋拍卖 / AIGC 讲师页——因为只看了目录名。

## 二、整体工作流概览（九阶段）

```
阶段 0: 对话引导          → 自然追问关键信息，不让用户填表
阶段 1: 深搜对方          → 多平台 OSINT，每条信息带来源
阶段 2: 对方画像          → 基本信息 + 关注点 + 决策位置 + 沟通风格
阶段 3: 推断需求          → JTBD/ACH/Empathy Map，有推理链
阶段 4: 嘉宾谱系（如适用）→ 对方若有节目/栏目/团队，必做
阶段 5: 翻用户资料        → 从对方需求倒推，深挖不扫表面
阶段 6: 制定策略          → 反差设计 + 确认升级反转 + 预判 Q&A
阶段 7: 输出              → markdown 简报 + 一句话摘要 + open 弹出
阶段 8: 会后复盘          → 修正档案、调整策略、记录判断对错
```

**每个阶段都不是线性的。** 阶段 1 搜到新信息，可能要回到阶段 0 追问用户。阶段 3 推断出需求后，可能发现阶段 1 搜得不够，要补搜。**允许回滚。**

## 三、阶段 0：对话引导流程

> **这是整个 skill 的入口。做得好不好，决定后面 7 个阶段的地基。**

### 3.1 入口识别

用户说以下话时，skill 被触发：

触发词（中文优先）：
- "帮我准备见 XXX" / "我要见 XXX"
- "我明天要去见一个人" / "下周有个见面"
- "约了某某" / "某某要来"
- "调研一下这个人"
- "见面策略" / "会面准备"
- "我去采访 / 被采访" / "上节目 / 上直播"
- "prep meeting with" / "meeting prep"

识别到触发词后，**不要**立刻开始搜索。先进入下面的对话引导。

### 3.2 第一轮问题（身份 + 场景）

**不是填表。是自然对话。问完一个，根据回答决定下一个。**

```
第一句开场（温度先铺好）：
"好的，我来帮你想透这次会面。先问你几个问题，然后我去深搜他。"

Q1（必问）：他是谁？
  - 真名、网名、社交账号、所在单位、职位
  - 不是要用户列清单，是"你能告诉我他的哪些信息？"

Q2（必问）：你们是怎么对上的？
  - 他主动找的你，还是你主动约的？还是第三方引荐？
  - 如果第三方引荐，是谁？引荐人的关系是？
  - 如果主动找的，他通过什么渠道 / 哪个作品找到你的？
  
Q3（必问）：这次会面的形式？
  - 线上 / 线下
  - 一对一 / 有其他人
  - 吃饭 / 会议室 / 咖啡 / 直播 / 录节目 / 上课
  - 预计时长

Q4（必问，但最关键）：你最想从这次会面中得到什么？
  - 不要接受"看看能聊点什么"这种答案
  - 追问："你最真实的期待是什么？上节目？拿资源？建立长期关系？还是就是认识一下？"
  - 如果用户说不清楚，给选项："A. 被采访 / 上节目 B. 求资源（融资/合作/引荐）C. 接商单（被求资源）D. 平等交流 E. 请教/拜访"
```

**关键**：Q4 的回答决定了整个策略的走向。如果用户说"不知道"，你必须追问到具体。

### 3.3 第二轮问题（根据会面类型分叉）

根据 Q4 的答案，进入不同的追问路径：

#### 分叉 A：被采访 / 上节目

```
A1: 他的节目/媒体是什么？频率？大概播放量级？
A2: 他之前找过你几次？聊到什么程度了？
A3: 有没有发过资料给你？（YouMind 链接？你的作品？节目大纲？）
A4: 他提过录制时间/形式了吗？还是这次是试镜/初聊？
```

#### 分叉 B：求资源（融资 / 合作 / 引荐）

```
B1: 具体想要什么？（多少钱？什么合作？引荐到谁？）
B2: 他有决策权吗？还是要向上汇报？
B3: 你有什么"可以交换"的东西？（你能给他什么）
B4: 他的替代选择是谁？（他不选你还有谁）
```

#### 分叉 C：接商单（被求资源）

```
C1: 他想要什么交付物？预算范围？时间线？
C2: 他为什么找你？（什么作品打动了他）
C3: 这是一次性还是长期？
C4: 有没有其他竞争者？
```

#### 分叉 D：平等交流（同行 / 校友 / 朋友）

```
D1: 你们的共同背景？
D2: 他最近在做什么？（如果用户知道）
D3: 你想从这次交流建立什么？（朋友？合作伙伴？长期互相助攻？）
```

#### 分叉 E：请教 / 拜访

```
E1: 他的专长是什么？
E2: 你最想请教的具体问题？
E3: 你能给他什么作为回报？（哪怕只是好内容、故事、人脉）
```

### 3.4 第三轮问题（搜索前的最后确认）

在开搜之前，告诉用户你要做什么：

```
"好，我现在去深搜他。搜的范围：
 - X/推特（看他最近在关注什么）
 - [如果是媒体人]他的节目/栏目（过往嘉宾和风格）
 - 公众号/36氪/LinkedIn（公开履历）
 - GitHub（如果他提技术）
 - 视频平台（如果他做内容）

大概需要 [X] 分钟。搜完我会汇报关键发现，我们一起判断。"
```

**关键**：告诉用户要多久，管理期待。搜完后**必须先汇报**，不要闷头出报告。

### 3.5 对话中的判断信号

用户的回答里有这些信号时，要特别注意：

| 用户说 | 信号 | 你该做 |
|---|---|---|
| "随便聊聊" | 低期待或想隐藏真实需求 | 温和追问："那如果真能聊出点什么，最想得到什么？" |
| "他人挺好的" | 用户已经有好感偏见 | 提醒自己：搜的时候要客观，不要也戴好感滤镜 |
| "我觉得他可能..." | 用户在猜测 | 把用户的猜测记下来，但**必须**去搜验证 |
| "我听说他..." | 二手信息 | 标注"来源：用户听说"，去搜原始出处 |
| "他找我好几次了" | 他有持续兴趣 / 用户一直没给确定答复 | 追问历史互动详情，可能有未解决的旧信号 |
| 语气焦虑 | 用户对这次会面压力大 | 策略上要增加"降低决策成本"的设计 |
| 语气轻松 | 用户已经成竹在胸 / 轻敌 | 温和提醒可能的盲区 |

### 3.6 避免的对话模式

**❌ 不要一次性问 10 个问题。** 问 3-4 个，根据回答再追问。

**❌ 不要把问题列成 markdown 表格让用户填。** 用自然语言一句一句问。

**❌ 不要在用户还没说完就开始搜。** 先听完，判断够不够再决定。

**❌ 不要假装听懂了。** 如果用户的回答有歧义，立刻追问："你说的 X 是指 Y 还是 Z？"

**❌ 不要把搜索过程变成演讲。** 用户说完他那侧，立刻进入搜索，不要长篇复述。

## 四、阶段 1：深搜对方 — 平台技术底层（最重要的一章）

> **这一章是整个 skill 的技术地基。每个平台都写清：有专用工具时怎么用，没有时如何 fallback。**

### 4.1 搜索前的规划（CIA Intelligence Cycle）

在开搜之前，明确三件事：

1. **我要知道什么？**（Planning）
   - 他是谁（基本事实层）
   - 他最近在关注什么（行为信号层）
   - 他的决策位置（KPI 层）
   - 他的沟通风格（人格层）

2. **在哪搜？**（Collection）

### 4.15 搜索的反模式（不要做什么）

- ❌ **不要只搜一个平台就下结论**。X 和公众号可能呈现完全不同的人。
- ❌ **不要忽略时间**。三年前的立场和现在可能完全相反。
- ❌ **不要只看原创内容**。回复、转发、点赞更诚实。
- ❌ **不要相信自我介绍**。Bio 是表演，行为才是真相。
- ❌ **不要漏了"沉默领域"**。他从不提什么 = 他回避/不懂的地方。
- ❌ **不要搜完就开始出报告**。先回到用户面前汇报关键发现，确认推断方向。
- ❌ **不要搜到什么都记**。按"对方需求推断"这个目的筛选，无关的不存。

### 4.16 搜索耗时预算

典型一次深搜，时间分配：

| 平台 | 时间 | 说明 |
|---|---|---|
| X | 3-5 分钟 | 最重要的单一来源 |
| 媒体文章 | 3-5 分钟 | 36氪 / 公众号 / LinkedIn |
| 节目/内容 | 5-10 分钟 | 如果对方是媒体人，花这里最多 |
| GitHub | 2-3 分钟 | 如果对方是开发者 |
| 其他平台 | 2-5 分钟 | 视对方画像 |
| 整理归档 | 5 分钟 | 结构化存储 |

**总预算：20-30 分钟。** 超过就要警惕——是不是在无关细节上浪费了？关键推断拿到没有？

**优先级从高到低：**

1. **用户的固定档案**

   - `~/.claude/skills/PAI/USER/TELOS/CHRONICLES.md`（人生节点，**故事素材金矿**）
   - `~/.claude/skills/PAI/USER/TELOS/GOALS.md`（年度目标）
   - `~/.claude/skills/PAI/USER/ABOUTME.md`（性格/协作方式）

2. **用户电脑里的项目目录**
   - `~/projects/`（正式项目）
   - `~/Desktop/`（正在进行的 / 随手放的）
   - `~/Documents/`（长期文档）

3. **用户的在线资产**
   - GitHub 账号（用 `gh repo list` / API 查 star 数）
   - 个人网站 / 作品集

4. **历史会面档案**（累积关系图谱）
   - `~/.aoe/meeting-secretary/contacts/<人名>/profile.md`（如果之前见过）
   - `~/.aoe/meeting-secretary/contacts/<人名>/meetings/*.md`（所有往期记录）

### 8.2 深挖的标准操作

**❌ 错误做法：** 只看目录名，看到 `~/Desktop/xxx/` 就下结论"这里有 xxx 资料"。

**✅ 正确做法：**

#### 8.2.1 扫目录 → 进子目录 → 读文件

```bash
# 第一步：用 Glob 找所有相关目录
Glob: ~/Desktop/**/*<关键词>*
Glob: ~/projects/**/README.md

# 第二步：对每个有希望的目录，Glob 找文件
Glob: <目录>/**/*.pdf
Glob: <目录>/**/*.pptx
Glob: <目录>/**/*.md
Glob: <目录>/**/*.html

# 第三步：Read 打开文件看内容
Read: <具体文件路径>
```

#### 8.2.2 PDF / PPTX 怎么读

**PDF：**
```
Read tool 直接支持 PDF（前 2000 行或指定页数）
对大 PDF：Read 指定页数范围 pages: "1-10"
```

如果 Read 不支持某类 PDF：
```bash
# 用 pdftotext / pdftk
pdftotext <文件.pdf> - | head -500
# 或者用 Bash 调 Python:
python3 -c "import pypdf; r=pypdf.PdfReader('<文件>'); print(r.pages[0].extract_text())"
```

**PPTX：**
```bash
# 直接用 unzip（.pptx 就是 zip）
unzip -p <文件.pptx> ppt/slides/slide1.xml | xmllint --xpath "//*[local-name()='t']/text()" -
```

或者 Bash 脚本：
```bash
for i in $(seq 1 50); do
  unzip -p <file.pptx> ppt/slides/slide$i.xml 2>/dev/null | \
    grep -oP '<a:t[^>]*>\K[^<]+' | \
    tr '\n' ' '
  echo
done
```

**Word / .docx 同理。**

#### 8.2.3 HTML 怎么读

```bash
# 直接 Read
Read: <文件.html>

# 如果 HTML 太乱，提取文字
cat <文件.html> | pup 'body text{}'
# 或者
cat <文件.html> | lynx -stdin -dump
```

#### 8.2.4 按关键词扫整个项目

```bash
# 找所有提到 "某关键词" 的文件
Grep: <关键词> path: ~/Desktop/xxx

# 查 markdown 里的所有标题
Grep: "^#" glob: "*.md" output_mode: content
```

#### 8.2.5 GitHub 数据核查（必做）

**不要相信目录名里的 README**。去 GitHub 查实际 star 数：

```bash
# 用户所有 repo 按 star 排序
gh repo list <用户> --limit 100 --json name,stargazerCount,description,primaryLanguage \
  | jq 'sort_by(-.stargazerCount)'

# 总 star 数
gh repo list <用户> --limit 100 --json stargazerCount \
  | jq '[.[].stargazerCount] | add'
```

**真实案例警示**：某次准备时，第一轮只扫目录完全没发现用户 GitHub 有 142 star 的项目——因为目录名里没体现 star 数，必须去 GitHub 查。

#### 8.2.6 本地文件 + 在线数据交叉验证

用户电脑里可能有老版本 CV 说"GitHub 100 stars"，但实际去 GitHub 查是 300+。**相信在线数据**，更新本地认知。

### 8.3 按"对方需求"筛选

**不是把用户有的全列出来。** 翻资料时带着阶段 3 的需求推断做筛选。

**筛选问题**：
- 这条素材能不能回应对方的功能性需求？
- 这条素材能不能让对方的情感需求被满足？
- 这条素材对对方向上汇报时有没有说服力？

**举例**：
- 对方需要"能出彩的节目嘉宾" → 选有冲击力、反差、可验证的素材

- 对方需要"快速交付" → 选展示过往准时交付案例

### 8.4 筛选三关（铁律 2）

每条弹药过三关：

| 关 | 问题 | 判断标准 |
|---|---|---|
| 第一关 | 在对方的场景里能震住人吗？ | 模拟对方反应 |
| 第二关 | 做这事的人多不多？ | 中国做这件事的大概有多少人？满大街有就砍 |
| 第三关 | 和对方的需求匹配吗？ | 能不能回答对方某个具体需求 |

**不过关就砍，哪怕听起来很厉害。**

### 8.5 找两个极端（铁律 3）

```markdown
- 证据：<文件路径 / URL>
- 震撼点：<为什么能震住>

### 反差的力量
```

## 九、阶段 6：制定展示策略

> 到这里你已经有：对方画像 + 对方需求推断 + 嘉宾谱系（如适用）+ 两个极端的弹药。**现在要把这些变成一次具体会面的剧本。**

### 9.1 核心矛盾（教员矛盾论）

每次会面都有一个"主要矛盾"——**抓住它 = 把所有准备集中在这一点上**。

```markdown
## 核心矛盾

### 这次会面的主要矛盾是什么？
<一句话>

### 不是什么
<常见误解：这次会面的矛盾看起来是 X，但其实不是>

### 用户该集中发力的点
<针对主要矛盾的一个关键动作>
```

**举例**：
- 看似矛盾："Jane 能不能上节目？"
- **真正矛盾**："这一期凭什么跟前面 10 期不一样？"
- 集中发力：不要证明"我合格"，要证明"我不同"

### 9.2 展示节奏：确认 → 升级 → 反转

**核心节奏模式**（来自实战验证的展示心理学）：

```
阶段 1：确认（强化对方已有的好感/认知）
  目的：让对方确信"我对他的第一印象是对的"
  手段：用对方最初被吸引的那个点，证明"不是偶然"
  心理效应：承诺一致性（Cialdini）——对方强化对你的正面判断

阶段 2：升级（把好感升级为"比我以为的还厉害"）
  目的：让对方觉得"我低估了他"
  手段：用第三方背书（奖项/拍卖/媒体/数据）
  心理效应：社会认同（Cialdini）——第三方认可比自我宣称有力 10 倍

阶段 3：反转（给一个完全意料之外的面）
  目的：打破对方的预设，建立新品类认知
  手段：用另一个极端，最好是完全不相关的领域
  心理效应：差异化记忆——反差大的信息被大脑标记为"重要"
```

**铁律**：**不要一开始就扔反差。** 先让对方确信第一印象是对的，再给他意外。

**举例**：
```
确认 → 《初始方程》完整系列 PDF（他来的原因，视觉震撼）
     → "这组作品进了荣宝斋拍卖"（一句话，看他反应）
     → "去年参了重庆国际光影节，实体展"（再加一句）

升级 → 荣宝斋展览材料 + 光影节合同 + IES 照明奖
     → 让"好看"升级为"被顶级认可的好看"

反转 → "但你可能不知道的是——我在 GitHub 上有 6 个开源项目，300 多个 star"
     → 打开 github.com/<handle> 给他看
     → "我不会写代码。这些全是用 AI 做的。"
     → 让他自己消化这个反差
     → 追问方向 = 他认为观众最感兴趣的方向
```

### 9.3 Voss Accusation Audit — 指控审计（FBI 谈判术）

**最常被忽略但最关键的一步**：列出对方可能对用户说的最难听的话 / 最大的顾虑，提前化解。

```markdown
## 指控审计

### 对方心里可能的顾虑（即使不说出来）
1. "这个人是不是只是在蹭 AI 热度？" 
   - 化解方式：<用具体事实>
   
2. "她的作品是不是用同一套模版批量生产？"
   - 化解方式：<展示风格跨度 / 创作过程>

3. "她上了节目会不会只讲表面，没干货？"
   - 化解方式：<准备技术细节备用 / 案例深度>

4. "她会不会一上节目就打广告？"
   - 化解方式：<事先说明你的分享原则>

### 用 Label 提前化解
在合适的时机主动说出对方心里的顾虑：
"你可能会想——[顾虑]。其实我也担心这个，所以我准备了..."

这叫 "calling out the elephant in the room"——
不回避反而提升信任。
```

### 9.4 Cialdini Pre-Suasion 预说服

在最关键信息抛出之前，先铺垫让对方进入"接受状态"：

```markdown

- 我和对方的共同点：<背景 / 观点 / 经历>
- 在开场 3 分钟内如何带出这个共性？
- 目的：让对方下意识觉得"我们是一类人"

### Privileged Moment（注意力最集中的时刻）
- 对方最专注的时刻是：<预判>
- 在这个时刻传递什么核心信息？

### 请教（不是征求意见）
- 问题：<一个关于他专长领域的请教>
- 心理效应：请教让对方心理上向你靠拢（Benjamin Franklin Effect）
```

**"这次会面怎么才能搞砸？"**

### 9.7 不该说的

**同等重要**。有些东西不主动说，除非对方问：

```markdown
## 不主动说（除非对方问）

### 技术细节类
- 系统怎么搭的 / 用了什么工具
- 原因：对方关心的是结果，不是工艺

### 商业/变现类
- 正在测试的商业模式（尤其是还没验证的）

- 原因：没验证的不说，说了减分

- 对某个同行的评价
- 原因：增加不必要的风险

- 创业艰难 / 情绪起伏
- 原因：不是每个人都愿意共情私人困难
```

### 9.9 现场的应变规则

```markdown

- 不评价，转话题
- 不要为了接话而发表立场
```

- 四种类型：行业观察者 / 产品实战者 / 逆袭者 / 学者专家
- Jane 最像类型 C（逆袭者），但多一个变量：**艺术**
- 不是在竞争同一个位置，是**全新品类**

### 15.3 两个极端的确定

经过深挖用户资料（PDF/HTML/GitHub API）：
- **极端 A**：荣宝斋拍卖（中国顶级拍卖行 + AI 作品）
- **极端 B**：GitHub 6 个开源项目 300+ stars（最高 142）
- **反差核心**：一个不会写代码的 45 岁女性

### 15.4 展示节奏设计

```
确认：《初始方程》完整系列 PDF → 荣宝斋拍卖 → 光影节实体展
升级：奖项（IES 照明奖 / 大地艺术节全球 20 佳 / TDC 2020）
反转：打开 GitHub，"我不会写代码"
```

### 15.5 筛选后砍掉的弹药

| 砍掉的 | 原因 |
|---|---|
| AI 培训师 | 中国满大街都有（铁律 2） |
| MBA 毕业 | 无区分度 |
| PAI 系统技术细节 | 对方不关心工艺 |
| 毛泽东思想学习 | 无关语境 |
| 10+ 个项目一次性列出 | 不是两个极端（铁律 3） |

### 15.6 铁律验证

这个案例在产生过程中，用户（Jane）多次纠正 AI：
- "你在从我出发列清单" → 铁律 1
- "AI 培训师满大街都有" → 铁律 2
- "你给我的四个点不痛不痒" → 铁律 4
- "你完全在迎合我" → 铁律 5
- 第一轮目录扫没发现 GitHub 300 stars → 铁律 7

**这些纠正直接沉淀为 `feedback_meeting_prep_rules.md`，然后固化进本 skill。**

## 十六、场景剧本库（五类会面完整剧本）

> 这一章给出五类典型会面的完整剧本。每个剧本包含：典型信号、分析侧重、展示节奏、话术片段、预判 Q&A、不该说的、会后动作。
> 实战使用时，skill 根据阶段 0 的 Q4 答案自动进入对应剧本。

### 16.1 剧本 A：被采访 / 上节目

#### A.1 典型信号
- 对方是媒体人（记者 / 主持人 / 节目制作人）
- 他主动约你（或通过引荐主动到你）
- 形式：采访 / 播客录制 / 直播 / 视频对谈

#### A.2 分析侧重
- **嘉宾谱系分析必做**（阶段 4）
- 节目的观众画像
- 节目的"爆款公式"（哪一期播放量最高，为什么）
- 主持人的提问风格（硬核 vs 轻松 vs 追问）
- 主持人的隐性议程（他想证明什么 / 反驳什么）

#### A.3 策略核心

> **不是"你能不能上"——是"这一期凭什么跟前面的不一样"。**

你的位置 = **节目的一个缺失维度**。不是和前面嘉宾比优秀，是填空白。

#### A.4 展示节奏（60 分钟采访）

```
0-5 min   温度 + Unity
  → 聊他最近发的内容，证明你看过
  → 带一句你对他节目的观察（但不是指点，是学习的角度）

5-15 min  确认（他为什么被你吸引）
  → 展示他最初看到的那个东西的完整版
  → 给一个第三方背书（奖项/拍卖/媒体），让他从"喜欢"升级到"不只是喜欢"

15-35 min 升级 + 反转
  → 中段注入一个意想不到的极端
  → 让作品说话，不自己吹
  → 准备一两个可录制的金句（能当标题）

35-50 min 深度 Q&A
  → 预判他会问的 5-8 个问题，每个有要点
  → 对尖锐问题用 Accusation Audit 提前化解

50-60 min 收尾
  → 明确下次动作（录制时间 / 发布安排）
  → 不要"回头联系"
```

#### A.5 话术片段

**开场破冰**：
> "你发的那期 [某节目] 我看完了，你问 [嘉宾] [问题] 那段我印象很深——因为 [具体观察]。所以今天能聊挺高兴。"

**确认节奏的第一句（打开你的作品后）**：
> "这是 [作品名]。[一句客观描述，不说'我做得好']。"
> 等他反应。

**升级（展示第三方背书）**：
> "这组 [进了 XX / 拿了 YY]。[一句事实，不夸耀]。"

**反转（打开另一个极端）**：
> "但你可能不知道，我 [完全不同领域的成就]。[一个反差事实]。"
> 停下来。让他消化。

**对金句的铺垫**：
> "我觉得 [一个反直觉观点]——[用一个具体案例解释]。"
> 金句必须短、可录、能当标题。

**收尾**：
> "这次聊得很好。下一步怎么走？[给他两个选项让他选]，我这边都可以配合。"

#### A.6 预判 Q&A（媒体采访通用 7 问）

| 他可能问 | 回答要点 |
|---|---|
| "你是怎么开始做 [X] 的？" | 时间线 + 一个关键转折，不要流水账 |
| "[作品名] 是怎么做出来的？" | 讲过程但强调"审美判断" vs "工具使用" |
| "普通人能学会吗？" | 能 + 关键不是技术是眼光 + 回扣金句 |
| "你怎么赚钱？" | 诚实说实验中的几条路，没验证的别吹 |
| "你跟其他 [同类] 有什么不同？" | 反差定位，两个极端 |
| "你用什么工具？" | 简单提，不展开技术细节 |
| "对行业你怎么看？" | 给一个有态度但不偏激的观点 |

#### A.7 不该说的（采访场景）
- 对同行的负面评价（会被剪成冲突点）
- 还没验证的商业数字（说了减分）
- 政治 / 宗教 / 敏感立场
- 你觉得对方节目"缺什么"（指点别人业务）

#### A.8 会后 24 小时
- 微信感谢 + 附上他提到想看的素材（立刻，不拖延）
- 如果聊到标题方向，发一版你建议的标题给他参考
- 如果他暗示了录制时间，直接发日历邀请

### 16.2 剧本 B：求资源（融资 / 合作 / 引荐）

#### B.1 典型信号
- 你主动约他（或通过引荐）
- 他是投资人 / 合作方决策者 / 有资源的前辈
- 你明确知道自己想要什么

#### B.2 分析侧重
- 他当前的**出手节奏**（最近投 / 合作了什么）
- 他的决策链（有没有合伙人 / 投委会 / 团队要说服）
- 他的"专业偏好"（赛道 / 阶段 / 金额）
- 他公开表达过的"避雷标准"（他说过什么情况绝对不投）
- 他的 BATNA（他在看的其他项目 / 合作方）

#### B.3 策略核心

> **不是证明你"优秀"——是证明你"不可替代" + "低决策成本"。**

投资人 / 合作方每天见太多"优秀"的人。他们真正买单的是：
- **不可替代性**：只有你能做成这件事 / 只有现在能做 / 只有这个价格能做
- **低决策成本**：不用他费劲说服别人 / 不用承担不必要的风险

#### B.4 展示节奏（30-45 分钟会议）

```
0-3 min   温度 + 你对他近期动向的观察（证明做过功课）

3-10 min  一分钟说清项目（SCR 框架）
  → Situation: 现状 / 市场
  → Complication: 问题 / 张力
  → Resolution: 我的方案 / 我的角色
  → 不要铺垫、不要"请允许我先介绍..."

10-20 min 证明不可替代性
  → 两个极端作为"你为什么是做这件事的合适人"
  → 具体数据 / 里程碑 / 已有案例

20-30 min 降低决策成本
  → 提前列出对方可能的顾虑（指控审计）
  → 每个顾虑用 Label 化解
  → 具体 Ask + 最小可接受版本

30-40 min 问他问题
  → 用 Calibrated Questions（What/How 开头）
  → "你觉得最大的挑战是什么？"
  → "我们怎么能更好地合作？"
  → 让他说话，你听

40-45 min 收尾
  → 明确下一步动作（不是他的 follow-up，是你的）
  → "这周五我发你 [具体材料]，下周二前你看一下，我们周三聊？"
```

#### B.5 话术片段

**一分钟说清（SCR）**：
> "[市场的 Situation]。但 [Complication]。我在做 [Resolution]——我是 [身份]，已经 [具体里程碑]。这次想和你聊 [具体 Ask]。"

**不可替代性的表达**：
> 不说："我们是市场第一"（虚）
> 说："过去 X 个月我们做了 Y，这件事我们比别人快 Z，因为 [具体原因]。"

**指控审计的预告**：
> "你可能会想——[顾虑]。我自己也担心这个，所以我们 [具体应对]。"

**具体 Ask**：
> "我想聊 [数额 / 规模 / 时间线]。如果这次不合适，最小的合作方式是 [次优选]。"

**Calibrated Questions**：
> "你看过这个赛道其他项目，你觉得我们最大的不同是什么？"（让他说出他的看法，比你自己说更有力）

#### B.6 预判 Q&A

| 他可能问 | 回答要点 |
|---|---|
| "你为什么做这个？" | 个人故事 + 痛点，不讲宏观 |
| "市场规模多大？" | 有数据，但要能算清楚（不要虚 TAM） |
| "你们的壁垒是什么？" | 三层：技术 / 数据 / 网络效应 |
| "竞争对手？" | 承认有，说明你为什么不同 |
| "需要多少钱做什么？" | 分阶段、具体里程碑、每阶段产出 |
| "退出路径？" | 有态度但不承诺 |
| "为什么是现在？" | 时机论据 + 你为什么在这个时点做 |

#### B.7 不该说的
- 夸大市场 / 用户数 / 收入（一定会被查到）
- 把竞争对手说一无是处（显得不成熟）
- "你放心，一定能成"（没人能承诺）
- 当场同意所有要求（留空间谈条款）

#### B.8 会后 24 小时
- 当天晚上发一封简短邮件 / 微信：感谢 + 他关心的几个问题的书面回答
- 附件：他提到想看的材料
- 明确下次时间和交付内容

### 16.3 剧本 C：接商单（被求资源）

#### C.1 典型信号
- 对方主动联系你，是**客户**（不是同行）
- 他有预算 / 需求 / 时间线
- 他在多选，你是候选之一

#### C.2 分析侧重
- 他的**业务类型**（ToB 品牌 / ToC 电商 / 广告 agency / 媒体平台）
- 他的决策链（谁拍板 / 谁验收 / 谁掏钱）
- 他的**过去合作模式**（看他们的其他供应商作品）
- 他的预算级别（通过行业水平 + 他们过往项目推算）
- 他的**真实痛点**（通常不是他说的那个）

#### C.3 策略核心

> **不是"我什么都能做"——是"在你这个需求下，我是最合适的"。**

客户最怕的是：
- 外包了出去但质量不可控
- 花了钱但不解决问题
- 沟通成本高

你要传递的是："**和我合作决策成本低 + 交付可预期 + 有真实案例证明。**"

#### C.4 展示节奏（30-60 分钟会议）

```
0-5 min   破冰 + 他公司的近期观察

5-15 min  先听，不急着展示
  → 让他完整说清需求
  → 用 Calibrated Questions 深挖
  → "你希望最终用户看到这个时有什么反应？"
  → "如果这次做得好，下一步你想做什么？"

15-30 min 针对他的需求展示案例
  → 不是作品集全倒
  → 只拿和他需求最相关的 2-3 个案例
  → 每个案例讲：客户当时的需求 → 我的方案 → 结果数据

30-45 min 报价 + 交付框架
  → 给报价区间不给单价（留谈判空间）
  → 清晰的交付里程碑
  → 沟通节奏（多久同步一次、谁是接口）

45-60 min 收尾
  → 明确下一步：提案什么时候出 / 合同什么时候签
```

#### C.5 话术片段

**让对方说清需求（Calibrated）**：
> "你最希望这次合作解决的**一个**最关键的问题是什么？"
> "你之前和其他人合作过类似项目吗？哪里最让你不满意？"

**针对性展示**：
> "基于你说的 [需求]，我想给你看一个类似的案例——[客户名] 当时的需求是 [X]，我们做了 [Y]，最终 [Z 数据]。"

**报价**：
> "基于你说的规模，这种项目市场价在 [低] 到 [高] 之间。我建议我们分两阶段——第一阶段 [金额 + 交付物]，验证方向对了再进第二阶段。这样你的风险最低。"

#### C.6 预判 Q&A

| 他可能问 | 回答要点 |
|---|---|
| "你做过类似的吗？" | 拿最相关的案例 |
| "价格能再谈吗？" | 不立刻让步，问他"预算在什么范围" |
| "你一个人做还是有团队？" | 诚实说 + 强调流程可控 |
| "时间能再快吗？" | 给两个选项：标准 vs 加急（不同价） |
| "如果效果不好怎么办？" | 分阶段 + 验收标准 + 修改次数 |
| "为什么选你不选 [竞品]？" | 给具体差异，不贬低对方 |

#### C.7 不该说的
- "多少钱都能做"（显得没定位）
- "我这周就能给你"（过度承诺）
- 前客户的负面信息（职业操守问题）
- 降价求合作（贬值自己）

#### C.8 会后 24 小时
- 24 小时内发书面提案（时间点很重要，拖延 = 不专业）
- 提案里明确：范围 / 时间 / 价格 / 验收标准 / 双方职责

### 16.4 剧本 D：平等交流（同行 / 校友 / 朋友）

#### D.1 典型信号
- 双方社会地位相当
- 没有明显的资源 / 交易诉求
- 可能由第三方撮合 / 行业活动认识
- 形式：咖啡 / 午餐 / 小规模 meetup

#### D.2 分析侧重
- 共同兴趣 / 共同背景（Unity Principle）
- 他最近在思考什么 / 纠结什么
- 你们可能的**互补性**（他的长处补你的短处 / 反之）
- 长期关系价值（而非短期交易）

#### D.3 策略核心

> **不是"展示你多厉害"——是"建立可长期续值的关系"。**

平等交流最容易失败的模式：
- 各自表演自己，没有真的交流
- 急于索取（下次能不能引荐 X）
- 没有后续（聊完就没了）

#### D.4 展示节奏（60-90 分钟咖啡）

```
0-10 min   共同点 / Unity
  → 找两个人的交集（学校 / 行业 / 兴趣 / 共同朋友）
  → 从这个点展开

10-30 min  你主动给价值
  → 分享你最近的发现 / 思考 / 资源
  → 不是炫耀，是"我想你会感兴趣这个"

30-50 min  真诚请教
  → 问他一个你真的好奇的问题
  → 听他说，不抢话

50-70 min  互相介绍资源 / 联系人
  → 如果合适，互相 intro 对你们都有价值的人

70-80 min  约定具体后续
  → 不是"以后多聊"
  → 是"下个月你去 [城市] 的时候我帮你约 [具体人]"
```

#### D.5 话术片段

**找共同点**：
> "你是 [学校] 的对吧？我也是。我记得 [某个共同场景]——"

**给价值**：
> "你最近在 [话题]，我刚看了一篇文章 / 一个案例——[具体]——我觉得你会感兴趣。"

**请教**：
> "你在 [领域] 做了这么久，我最近一直在想 [问题]，你怎么看？"

#### D.6 不该说的
- 夸张的自我展示
- 直接索取（除非关系已经很熟）
- 对共同朋友的负面评价
- 长时间独白（超过 3 分钟你就在演讲不是交流）

#### D.7 会后 48 小时
- 发一个"这次聊到的 [具体内容] 的后续思考"
- 如果约了给他引荐谁，立刻发起
- 建立周期性触达（不是频繁，是每 1-3 个月一次有意义的接触）

---

### 16.5 剧本 E：请教 / 拜访

#### E.1 典型信号
- 你主动约见
- 对方是明确的"前辈 / 专家"
- 你有具体的学习诉求

#### E.2 分析侧重
- 他的**专长领域边界**（哪些问题他真懂，哪些是蹭热度）
- 他最近在输出什么（文章 / 演讲 / 课程）
- 他在这个领域的**未解问题**（你可能能帮他想通的点）
- 他的时间价值（他时间多贵）

#### E.3 策略核心

> **不要浪费对方的时间。带着具体问题去，不是"想认识一下"。**

好的请教 = 对方说完后觉得"这次聊也有收获"。

#### E.4 展示节奏（30-45 分钟）

```
0-2 min   明确时间约定（"我们聊 30 分钟够吗"）
          → 显得专业 + 让对方有预期

2-5 min   背景铺垫（为什么找他）
          → "我看了你 [具体内容]，有 [具体问题]"
          → 不超过 2 分钟

5-25 min  你的具体问题
          → 一次一个，不堆叠
          → 用 Calibrated Questions

25-35 min 给他价值的反向输出
          → 分享你的独特视角 / 资源
          → 让他觉得"我也学到了"

35-40 min 约定下一步 / 感谢
          → 不是"以后多请教"，是"下次我做完 [X] 回来找你"
```

#### E.5 话术片段

**开场**：
> "谢谢你抽时间。我今天想请教三个问题——[列标题]。我们聊 30 分钟够吗？"

**问问题**：
> "我遇到的情况是 [具体]。我自己的想法是 [方案 A 或 B]。但我在纠结 [具体点]。你怎么看？"
>
> 注意：先说你自己的思考，他才知道该在哪一层回答。

**反向输出**：
> "对了，我最近在做 [X]，发现 [一个有趣的洞察]，分享给你，也许对你 [相关领域] 有用。"

#### E.6 不该说的
- "你随便讲讲都行"（浪费对方时间）
- "我什么都不懂"（让对方要从基础讲起，浪费）
- 把所有你有的问题一次倒出来（挑 1-3 个最关键的）

#### E.7 会后 24 小时
- 立刻感谢 + 总结"你教我最大的一个点"
- 如果你做了什么照他说的方向推进，过几周告诉他结果（这是最好的感谢方式）

### 16.6 剧本选择决策表

skill 根据阶段 0 的 Q4 答案自动路由：

| Q4 的答案 | 进入剧本 |
|---|---|
| "被采访 / 上节目 / 做访谈" | A |
| "融资 / 求合作 / 找资源 / 求引荐" | B |
| "他要买我的服务 / 我是供给方 / 接商单" | C |
| "就是认识一下 / 闲聊 / 同行" | D |
| "请教 / 学习 / 求建议" | E |

**如果用户答不清**：按"他发起还是你发起"和"谁是需求方"两个维度问：

```
他发起 + 他有需求 → 剧本 A（采访）或剧本 C（买服务）
你发起 + 你有需求 → 剧本 B（求资源）或剧本 E（请教）
双方都没明显诉求 → 剧本 D（平等交流）
```

## 十九、反模式陷阱（新手 AI 最容易犯的错）

> 这一章记录 skill 运行中**最容易犯的错**。每次运行时 AI 都要过一遍这个清单。

### 19.1 搜索阶段的陷阱

**陷阱 1：只搜一个平台就下结论**
- 症状：只抓了 X 就得出"他是做 AI 的"
- 纠正：至少交叉 2 个独立平台（Trace Labs Verify 原则）

**陷阱 2：把老信息当成当前状态**
- 症状：引用 3 年前的采访说他"现在在做 X"
- 纠正：标注每条信息的日期。近 6 个月的优先。

**陷阱 3：把 bio 当事实**
- 症状：LinkedIn 上自称"世界级专家"，AI 直接采信
- 纠正：自我介绍是表演，用行为验证

**陷阱 4：漏了"沉默领域"**
- 症状：只总结他常聊什么，忽略他从不提什么
- 纠正：他从不提的话题同样重要（禁区 / 回避 / 不懂）

### 19.2 分析阶段的陷阱

**陷阱 5：给结论不给推理链**
- 症状："他想找 AI 艺术家嘉宾"（单句结论）
- 纠正：结论必须带"依据 + 证据 URL + 置信度"

**陷阱 6：把一个强信号等同于多个弱信号的合力**
- 症状：他一条推文提了 AI，就推断他"all in AI"
- 纠正：要看**模式**，不是单点。近 30 天有 X 条相关推文 = 模式；1 条 = 噪声。

**陷阱 7：忽略反面证据**
- 症状：只看支持假设的证据，忽略反驳的
- 纠正：CIA ACH 明确要求"尝试否证"——每个假设都要列反面证据

**陷阱 8：过度解读**
- 症状：他点赞了一条推就推断"他认同 X 价值观"
- 纠正：点赞很廉价，要看重复的模式

### 19.3 策略阶段的陷阱

**陷阱 9：从用户出发列清单**
- 症状：打开用户资料库，把所有能找到的成就列一遍
- 纠正：**铁律 1**——先充分理解对方，再带着"他需要什么"去翻资料

**陷阱 10：堆数量而非找反差**
- 症状：列了 15 条弹药
- 纠正：**铁律 3**——两个极端 > 十个中等

**陷阱 11：列满大街都有的东西**
- 症状："他是 MBA / AI 培训师 / 创过业"
- 纠正：**铁律 2**——过三关，不过关砍掉

**陷阱 12：给空泛建议**
- 症状："你应该展示你的审美能力"
- 纠正：**铁律 4**——具体到打开什么文件、说什么话

**陷阱 13：迎合用户**
- 症状：用户说 A 就改 A，用户说 B 就改 B
- 纠正：**铁律 5**——先有独立判断，用户修正再调整

### 19.4 输出阶段的陷阱

**陷阱 14：只给路径不 open**
- 症状："简报已生成，路径：..."
- 纠正：必须 `open <file>` + `open -R <file>`

**陷阱 15：简报过长**
- 症状：生成 100 页的分析报告
- 纠正：简报应该是**会面前能快速过一遍**的，不是学术论文

**陷阱 16：简报没有具体话术**
- 症状：只有"展示你的作品"
- 纠正：有具体的"第一句说什么、打开哪个文件、说什么"

**陷阱 17：没有指控审计**
- 症状：只列了"我要说什么"，没列"他心里可能想什么"
- 纠正：Voss Accusation Audit 必做——提前化解顾虑

**陷阱 17A：装不知道（信息来源伪装）** 🔴 致命
- 症状：用户告诉 AI"对方是北体大毕业的"，AI 给用户写话术"听说你是北体大的..."
- 问题本质：用户本来就认识对方、本来就知道这些信息，用"听说 / 据说 / 好像"让用户对对方**假装不知道** = 让用户在对方面前显得虚伪
- 纠正：**所有用户告诉你的对方信息，在话术里默认"用户已知"**。不需要伪装信息来源。对方也知道用户知道。直接用"你 XX + 你 YY 挺有意思的..."这种对等语气
- **记住**：用户和对方是已有关系或即将建立关系的**对等人**，不是用户要扮演什么角色去接近对方

**陷阱 17B：给话术模板，不给洞察** 🔴 致命
- 症状：AI 在简报里大量写 "**要说**：XXX" / "**首选开场白**：'听说你...'" / "**金句**：'跟你跑半马一个原理...'"
- 问题本质：**用户不需要 AI 教她怎么说话**。用户自己会说、自己认识对方、自己有临场感。用户要的是**对方的核心痛点、真相、她听不到的视角**——用户基于这些自己发挥
- 纠正：
  - **删除**所有 "**要说**：XXX" / "**首选话术**" / "**金句**" / "**模板**" 类段落
  - **替换为**：
    - "**她在意什么**"（她耳朵会对什么打开）
    - "**她想听到的信号**"（具体内容方向，不是台词）
    - "**她真正在问什么**"（问题背后的真问题）
    - "**你要守住的底线**"（不要变成什么，而不是要说什么）
  - 展示节奏可以给（什么时候做什么动作、信号怎么识别），但**不能给台词**
- **铁律**：**给洞察，不给剧本**。用户是真人在真实场合，不是演员在演戏
- **自检**：简报里出现引号包裹的整句"台词"就要警惕——引号里的话 99% 应该删掉或改成观点方向

### 19.5 复盘阶段的陷阱

**陷阱 18：跳过复盘**
- 症状：见完就不回来了
- 纠正：skill 要在见面当天晚上 / 第二天主动触发一次"你昨天的会面怎么样？"

**陷阱 19：只记事实不记判断对错**
- 症状："聊了 1 小时，他说了 X 和 Y"
- 纠正：必须对比之前的预判，标对 / 错 / 部分对

**陷阱 20：复盘不更新档案**
- 症状：复盘做了，但 profile.md 没变
- 纠正：复盘的结果必须写回 profile.md，否则系统不会变强

## 二十、使用流程快照（把整个 skill 浓缩成一页）

### 20.1 完整流程图

```
用户说"我要见 XXX"
  │
  ├─ 阶段 0：对话引导
  │   └─ 问 Q1-Q4，根据 Q4 决定剧本分叉
  │
  ├─ 阶段 1：深搜对方
  │   └─ 按平台优先级搜，每条信息带 URL
  │
  ├─ 中途汇报：给用户 3-5 个关键发现，确认推断方向
  │
  ├─ 阶段 2：对方画像
  │   └─ 基本信息 + 最近关注 + DISC + OCEAN
  │
  ├─ 阶段 3：推断需求
  │   └─ JTBD + ACH + Empathy Map + BATNA
  │
  ├─ 阶段 4：嘉宾谱系（如适用）
  │   └─ 类型分类 + 用户在谱系的位置 + 独特价值
  │
  ├─ 阶段 5：翻用户资料
  │   └─ 深挖不扫表面 + 从需求倒推 + 三关筛选
  │
  ├─ 阶段 6：制定策略
  │   └─ 核心矛盾 + 两个极端 + 确认升级反转 + 预判 Q&A
  │
  ├─ 阶段 7：输出简报
  │   └─ markdown + 摘要 + open + open -R
  │
  └─ 阶段 8：会后复盘（见完后）
      └─ 事实 + 印证 + 修正档案 + 下次动作
```

### 20.2 一次完整运行的时间分配（典型案例）

| 阶段 | 时间 | 用户参与度 |
|---|---|---|
| 0. 对话引导 | 3-5 min | 高（答 4 问题） |
| 1. 深搜 | 20-30 min | 低（AI 独立） |
| 中途汇报 | 2-3 min | 高（确认方向） |
| 2. 画像 | 5 min | 低 |
| 3. 需求推断 | 10 min | 中（可能需确认） |
| 4. 嘉宾谱系 | 10 min | 低 |
| 5. 翻资料 | 10-15 min | 低 |
| 6. 策略 | 10 min | 中（讨论两个极端选哪两个） |
| 7. 输出 | 5 min | 低 |

**总：75-90 分钟**。用户的净投入时间：15-20 分钟（剩余都是 AI 独立工作）。

### 20.3 运行前的 checklist

```
[ ] 数据目录存在（~/.aoe/meeting-secretary/）
[ ] me/profile.md 存在（否则先初始化）
[ ] 基础工具可用（WebSearch / WebFetch / Glob / Read）
[ ] 知道对方的至少一个强标识（姓名 / handle / 公司）
[ ] 用户能回答 Q4（最想要什么）
```

## 结语

这个 skill 的核心不是工具链，是**一种思考方式**：
- 从对方出发，不从自己出发
- 用证据推断，不凭直觉猜测
- 找两个极端，不堆十个中等
- 做助手不做领导，给具体不给空泛
- 搜了才有发言权，深挖不扫表面

**工具会过时。** X 镜像会挂，wechat-fetcher 可能哪天改名。但这七条思考方式不会过时。

**这个 skill 存在的意义**：让用户每次见重要的人时，不再是一个人焦虑——而是有一个智囊在身边，和她一起把局势想透。

每见一个人，这个智囊就多懂她一点。每复盘一次，这个智囊就更准一些。**越用越强**——这不是一个月抛弃的工具，是一个终身陪伴的系统。
